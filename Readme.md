# 浏览器小游戏《躲避方块》需求文档

## 1. 项目简介

开发一个可直接在浏览器中运行的 2D 小游戏《躲避方块》。  
玩家通过键盘控制角色移动，躲避不断出现或下落的方块，尽可能获得更高生存时间或分数。

## 2. 项目目标

- 实现一个轻量、可玩、可迭代的网页小游戏原型。
- 不依赖任何前端框架，使用原生技术完成核心功能。
- 结构清晰，便于后续增加新玩法（如加速、道具、关卡）。

## 3. 目标用户

- 想快速体验小游戏的普通玩家。
- 用于前端基础练习或教学演示的开发者/学习者。

## 4. 核心玩法需求

### 4.1 玩家控制

- 玩家可通过键盘控制角色移动（方向键或 WASD）。
- 角色移动需限制在游戏区域内，不可超出边界。

### 4.2 障碍物机制（方块）

- 敌对方块按固定频率或随机频率生成。
- 方块从上向下移动（或按设定方向移动）。
- 随游戏进行可逐步提升难度（生成频率变快、速度提升）。

### 4.3 碰撞与失败判定

- 当玩家角色与任意方块发生碰撞时，游戏结束。
- 结束后显示“游戏结束”提示和最终得分/生存时间。

### 4.4 得分系统

- 基础规则：存活时间越长，分数越高。
- 实时显示当前分数。
- 可选：记录本局最高分（当前会话）。

### 4.5 游戏状态流转

- 初始状态：显示开始提示（如“按空格开始”）。
- 进行中：持续刷新画面并处理输入、移动、碰撞、计分。
- 结束状态：显示结果并支持重新开始。

## 5. 非功能需求

- 运行流畅：普通电脑浏览器下动画不卡顿（目标 60 FPS，允许波动）。
- 代码可读：HTML、CSS、JS 分文件管理，命名清晰。
- 兼容性：主流现代浏览器可正常运行（Chrome、Edge）。

## 6. 技术选型

- 前端：HTML + CSS + JavaScript（无需框架）
- 开发工具：Cursor IDE
- 运行方式：直接用浏览器打开 `index.html`

## 7. 目录建议

```text
cursor-game-demo/
├─ index.html      # 页面结构与游戏容器
├─ style.css       # 样式与布局
├─ game.js         # 游戏主逻辑
└─ Readme.md       # 项目说明与需求文档
```

## 8. 验收标准（MVP）

- 打开 `index.html` 可直接进入游戏页面。
- 玩家可通过键盘移动角色。
- 方块会持续生成并移动。
- 碰撞后能正确结束并显示结果。
- 支持重新开始游戏。

## 9. 后续可扩展方向

- 增加暂停/继续功能。
- 增加移动端触屏控制。
- 增加音效、背景音乐、特效。
- 增加难度等级与排行榜（本地存储）。
